# 프론트엔드 서버(Frontend Server) 구현 태스크 체크리스트

백엔드 8단계 Phase에 맞춘 상세하고 구체적인 프론트엔드 UI 및 연동 태스크 목록입니다.

---

## [Phase 0] 프로젝트 초기 설정 및 기획
- [o] Vite 기반 Vue 3 (JavaScript) 프로젝트 생성
- [o] 라우터 (Vue Router) 및 상태 관리 (Pinia) 설정
- [o] API 통신을 위한 Axios 설치 및 기본 설정
- [o] 글로벌 디자인 토큰 (CSS 변수) 및 1:1 이미지 최적화 기본 스타일 (reset) 셋업
- [o] 공통 Layout 구성 (상단 헤더 고정, 반응형 분기)

## [Phase 1] 백엔드 기반 체계 (회원/인증/상점/공통)

  - [o] 로그인/회원가입 폼 (`Auth.vue`) 최신 엔티티(email, password, nickname)로 간소화

  - [o] 내 프로필 (`MyPage.vue`) 포인트/캐시 제거 및 새 포맷 렌더링 수정
  - [o] 비로그인 접근 제어용 라우터 네비게이션 가드 구현 및 전역 에러 바운더리(Toast 알림) 적용
- [o] **배송지 정보 연동**
  - [o] `Address.vue` 배송지 목록 조회 UI
  - [o] 주소 검색 및 추가/수정 모달 폼 구현
  - [o] 기본 배송지 설정 라디오버튼/상태 적용
- [o] **상점 (Store) 관리 및 공개 프로필**
  - [o] `StoreProfile.vue` 내 상점 편집(소개글, 프로필 이미지 등) UI 구현 (`PATCH /member/me` 연동)
  - [o] 상점 등급, 리뷰개수 등에 특화된 Badge 컴포넌트 개발
  - [o] 타 사용자에게 노출되는 공개 상점(Store) 페이지 UI 뼈대 준비

## [Phase 8] 성능 최적화 / 프론트 연동 / 검증
- [o] **데이터 로딩 시 Skeleton UI 플러그인 연동**
  - [o] 도감 탐색(`Dex.vue`) 시 시각적 안정성을 위한 Skeleton UI 구현 및 적용
- [ ] **실운영 마이그레이션 워크스루 점검**
  - [ ] 회원가입(인증) -> 메인 검색 -> 도감 연동 -> 실물 출품 -> 주문서 작성 -> 모의 결제 -> 송장 입력 -> 구매 확정 -> 판매 정산 -> 유저 평점 부여로 이어지는 전 과정 화면 전환 및 에러 부재시나리오 앤드투앤드(E2E) 점검
